## What’s Changed

First stable release of `simple-modern-pnpm` as a Copier template.

### Highlights

- Clarified the supported Copier workflows for:
  - starting a brand-new repo
  - rendering into a scratch directory
  - adopting the template into an existing TypeScript repo
  - pulling future updates with `copier update`
- Tightened the agent-friendly positioning and mirrored working guidance into generated
  `AGENTS.md` and `CLAUDE.md`.
- Replaced the ambiguous local full-check path with `pnpm check`.
- Made bootstrap safe before `git init` by deferring hook installation to `pnpm prepare`
  when needed.

### Template Correctness

- Removed admin-only leakage from generated repos, including the broken
  `compile-template` script and irrelevant `template/**` / `attic/**` config entries.
- Restored committed `.copier-answers.yml` support so `copier update` works cleanly.
- Synced the compiled `template/` output with the compile step.
- Normalized generated workflow formatting so fresh renders are formatting-clean.

### Validation

- Added an admin-only rendered-template smoke test that covers:
  - fresh render
  - install-before-`git init`
  - `copier update`
  - retrofit of a flat legacy repo into the generated workspace layout

### Maintainer Workflow

- Documented the template release process in `admin/releasing.md`.
- Clarified that Copier follows Git tags by default, so template tags are the release
  source of truth for consumers.
- Guarded the generated-project release workflow so tagging this template repo does not
  try to publish the placeholder package.
